# KDoc 与 Kotlin 文档写法

这个文件只负责 Kotlin/KDoc 的语言层写法和常见取舍。

仓库里哪些类型、接口方法或边界 DTO 必须补 KDoc，以 `avalon-code-commenting` 技能为准；这里不重复定义强制范围。

## 1. KDoc 应该说明什么

- 这个类型 / 函数解决什么问题
- 调用方最该知道的前置条件和后置语义
- 返回值的含义，尤其是 `null`、空集合、异常或 `Result` 的边界
- 关键副作用、线程或顺序要求
- 典型用法和必要示例

## 2. 写法建议

- 第一段先给高层语义，不要直接重复函数签名。
- 参数、返回值和异常说明以“补充信息”为主，不要逐字重述名称。
- 尽量用链接把说明连到相关类型和函数。
- 示例要能代表真实用法，而不是只写最小伪代码。

## 3. KDoc 细节

- KDoc 使用 Markdown 风格内联标记。
- 可以用方括号链接到代码元素。
- `@deprecated` 标签不是 KDoc 的推荐方式；弃用请使用 `@Deprecated` 注解。
- 需要生成文档时，优先考虑使用 Dokka。
