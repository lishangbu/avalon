# Outbox 决策树

## 1. 这些场景应当使用 Outbox

- 一次成功的本地提交之后，必须通知另一个 bounded context
- 一次成功的本地提交之后，必须延后发布集成事件
- 丢失这条事件会导致下游视图不一致

## 2. 这些场景通常不要上 Outbox

- 整个操作完全停留在单个上下文内，并在一个事务内结束
- 没有提交后的通知需求
- 操作只是纯查询或 projection 读取

## 3. 快速判断问题

按顺序问自己：

1. 是否存在业务写入
2. 成功后是否必须通知其他上下文或外部系统
3. 提交后如果发布失败，是否会造成正确性问题

如果答案是 `yes / yes / yes`，默认使用 outbox。
