# 注释模板

模板只提供结构，不要求逐句照抄。保留高信息量，删掉空话。

## 1. 模块入口文件

声明级说明默认使用 KDoc 风格 `/** ... */`。以下模板中的块注释都应按 KDoc 风格书写。

```kotlin
/**
 * <模块名> 负责 <核心职责>。
 *
 * 当前阶段它拥有：
 * - <能力一>
 * - <能力二>
 *
 * 它不负责：
 * - <不属于本模块的职责>
 *
 * 对外协作时，<其他上下文或模块> 应通过 <读取方式 / 契约类型> 接入，
 * 不直接依赖 <不应暴露的实现细节>。
 */
```

## 2. `shared-kernel` 公共抽象

```kotlin
/**
 * <类型名> 用来表达 <稳定的领域基线概念>。
 *
 * 这个抽象放在 `shared-kernel`，是因为：
 * - <跨上下文稳定性原因>
 * - <不依赖具体基础设施或单一上下文语义>
 *
 * 它不负责：
 * - <不应被误解承担的职责>
 */
```

## 3. 接口与公共契约

```kotlin
/**
 * <接口名> 定义 <能力边界>。
 *
 * 设计上把它抽成契约，是因为：
 * - <跨层 / 跨模块 / 测试替换 / 稳定 API 的原因>
 * - <为什么不直接暴露具体实现>
 *
 * 它不负责：
 * - <不属于该契约的职责>
 */
interface <InterfaceName> {
    /**
     * 执行 <方法目标>。
     *
     * 设计思路：
     * - <为什么提供这个方法>
     * - <调用方最容易误解的边界或约束>
     *
     * @param <param1> <参数一含义、取值约束或前置条件>
     * @param <param2> <参数二含义、是否允许为空 / 空集合 / 默认行为>
     * @return <返回值语义，以及空值 / 空集合 / 失败时的约定>
     * @throws <ExceptionType> <仅在异常语义本身对调用方重要时说明>
     */
    suspend fun <methodName>(<param1>: <Type1>, <param2>: <Type2>): <ReturnType>
}
```

如果返回 `Unit`，正文里要说明副作用、状态变化或输出结果，`@return` 可以省略。

## 4. 对外 Request / Response DTO

```kotlin
/**
 * <DTO 名称>，用于 <接口 / 事件 / 快照> 的 <请求 / 响应> 边界。
 *
 * 设计思路：
 * - <为什么需要这个边界类型，而不是直接暴露领域模型>
 * - <调用方最需要理解的契约约束>
 *
 * @property <field1> <字段一的作用、是否允许为空、默认行为或格式约束>
 * @property <field2> <字段二的作用，以及它和其他字段的关系>
 */
data class <DtoName>(
    val <field1>: <Type1>,
    val <field2>: <Type2>,
)
```

如果某个字段只是同类字段中的普通审计信息，例如 `createdAt`、`updatedAt`、`version`，
可以在类级 KDoc 中合并说明，不必为了凑覆盖率写重复废话。

 * - 事务范围：<说明>
 * - 输入 / 输出：<说明>
 * - 不在这里处理：<说明>

```

```
